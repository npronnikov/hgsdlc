# Генерация уточняющих вопросов в формате human-form

## Цель

Снять неопределённость запроса до начала реализации: сформулировать
вопросы, ответы на которые напрямую влияют на выбор решения.
Результат — интерактивная форма, которая откроется в gate UI и позволит
пользователю выбрать варианты ответов без ручного редактирования файлов.

---

## Шаг 1 — Анализ запроса

Прочитай запрос пользователя и определи:
- Что именно запрашивается?
- Какие части системы затронуты?
- Где наибольшая неопределённость — бизнес-правила, scope, технические решения?

Не изучай кодовую базу. Вопросы — только по запросу пользователя.

---

## Шаг 2 — Формулировка вопросов

Сформулируй от **3 до 8 вопросов**. Каждый вопрос должен:
- Быть конкретным и напрямую связанным с запросом
- Иметь 2-4 взаимоисключающих варианта ответа (type: radio или select)
- Или быть открытым (type: textarea) — только если вариантов нет

Типы вопросов по ситуации:
- `radio` — выбор одного варианта из 2-4 (большинство вопросов)
- `select` — выбор из длинного списка (5+ вариантов)
- `textarea` — открытый ответ (не более 1-2 вопросов)
- `checkbox` — множественный выбор (если возможны несколько ответов)

---

## Шаг 3 — Создание questions.form.json

Создать файл `questions.form.json` в рабочей директории запуска.

Строго соблюдать следующий формат:

```json
{
  "kind": "human-form",
  "version": "1",
  "title": "Уточняющие вопросы",
  "fields": [
    {
      "id": "field_1",
      "type": "radio",
      "label": "Текст вопроса?",
      "required": true,
      "options": ["Вариант А", "Вариант Б", "Вариант В"],
      "value": null
    },
    {
      "id": "field_2",
      "type": "textarea",
      "label": "Открытый вопрос (уточни детали)?",
      "required": false,
      "value": null
    },
    {
      "id": "field_3",
      "type": "checkbox",
      "label": "Что из перечисленного нужно включить?",
      "required": false,
      "options": ["Пункт 1", "Пункт 2", "Пункт 3"],
      "value": null
    }
  ]
}
```

**Правила:**
- `kind` — всегда `"human-form"`, `version` — всегда `"1"`
- `title` — осмысленный заголовок, отражающий тему вопросов
- `id` — уникальный snake_case идентификатор для каждого поля
- `value` — всегда `null` (заполнит пользователь)
- `required: true` — только для вопросов, без ответа на которые нельзя продолжить
- `options` — только для `radio`, `select`, `checkbox`; не использовать для `text`/`textarea`
- Варианты в `options` — конкретные, реалистичные, взаимоисключающие

---

## Ограничения

- Не писать код реализации
- Не изучать кодовую базу — только запрос пользователя
- Не задавать вопросы ради количества — только те, что реально влияют на решение
- Вывод — только файл `questions.form.json`, ничего кроме него
